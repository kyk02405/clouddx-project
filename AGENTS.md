# Repository Agent Rules

## Dev Logs (Mandatory)

When creating or editing any file under `docs/dev_logs/**`, you must:

1. Read `docs/dev_logs/DEV_LOGS_GUIDE.md` first.
2. Follow the required section/template rules in that guide.
3. Use a human name for `- 작업자:` (no tool/system names such as `CloudDX`, `Codex`, `AI`, `Assistant`).
4. Keep the log in UTF-8 and place screenshots under `docs/dev_logs/screenshots/YYYY-MM-DD/`.

If the guide and an existing log conflict, follow the guide and update the log format accordingly.
