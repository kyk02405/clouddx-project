# 🔧 Troubleshooting Report (2026-02-19)

## 이슈 1: K8s 매니페스트 오프라인 작업 전략 수립

### 📝 상황

- **환경**: 자택 (학원 K8s 클러스터 네트워크 접근 불가)
- **문제**: `kubectl` 명령어 실행 불가 (VPN/직접 접속 없음)
- **영향**: Phase 1 K8s 클러스터 설정 작업 진행 불가

### ✅ 해결 전략 (오프라인 준비)

- 모든 K8s YAML 매니페스트를 로컬에서 미리 작성
- `k8s-manifests/` 디렉토리 구조로 체계적으로 정리
- 각 단계별 README.md에 실행 순서 명시
- 학원 네트워크 접속 시 순서대로 `kubectl apply` 예정

### 📚 Lessons Learned

- 네트워크 제약 환경에서도 GitOps 방식으로 사전 준비 가능
- YAML 파일을 단계별로 분리하면 순차 적용이 용이함

## 이슈 2: Harbor Secret (imagePullSecret) 설정

### 📝 증상

- **문제**: K8s Pod가 Harbor 프라이빗 레지스트리에서 이미지를 pull할 수 없음
- **예상 에러**: `ErrImagePull: unauthorized: authentication required`

### 🔍 원인 분석

- K8s는 프라이빗 레지스트리 접근 시 `imagePullSecrets` 설정 필요
- `docker login` 정보를 K8s Secret으로 변환해야 함

### ✅ 해결 방법

```bash
# base64 인코딩된 docker config 생성
kubectl create secret docker-registry harbor-secret \
  --docker-server=192.168.56.12:8080 \
  --docker-username=admin \
  --docker-password=Himedia123! \
  --namespace=tutum-app
```

또는 `base/backend/secret.yaml`의 `harbor-secret` 직접 적용:

```bash
kubectl apply -f k8s-manifests/base/backend/secret.yaml
```

### 📚 Lessons Learned

- 프라이빗 레지스트리 사용 시 `imagePullSecrets` 설정은 필수
- Secret을 YAML로 관리하면 GitOps 파이프라인에 통합 가능

## 이슈 3: Kyverno Cosign 정책 - 공개키 미설정

### 📝 상황

- `kyverno/cosign-verify-policy.yaml` 작성 완료
- 실제 Cosign 키 쌍 미생성 상태

### ✅ 해결 절차 (학원에서 실행 예정)

```bash
# 1. Cosign 키 쌍 생성
cosign generate-key-pair

# 2. cosign.pub 내용을 policy YAML에 붙여넣기
# kyverno/cosign-verify-policy.yaml의 publicKeys 필드 수정

# 3. 이미지 서명
cosign sign --key cosign.key 192.168.56.12:8080/tutum/backend:latest

# 4. 정책 적용
kubectl apply -f k8s-manifests/kyverno/cosign-verify-policy.yaml
```

### 📚 Lessons Learned

- Kyverno 정책 적용 전 반드시 Cosign 키 쌍 생성 및 이미지 서명 선행 필요
- `validationFailureAction: Enforce` 상태에서 미서명 이미지 배포 시 Pod 생성 차단됨

## 이슈 4: Grafana 대시보드 자동 프로비저닝 경로 설정

### 📝 상황

- `grafana/provisioning/dashboards/dashboards.yml`에서 경로 설정 필요
- Docker Compose 볼륨 마운트와 경로 일치 확인 필요

### ✅ 확인 사항

`docker-compose.yml`의 Grafana 볼륨 설정:

```yaml
volumes:
  - ./grafana/provisioning:/etc/grafana/provisioning
```

`dashboards.yml`의 경로:

```yaml
options:
  path: /etc/grafana/provisioning/dashboards
```

→ 컨테이너 내부 경로 `/etc/grafana/provisioning/dashboards`에 JSON 파일 위치 확인 필요

### 📚 Lessons Learned

- Docker 볼륨 마운트 경로와 Grafana 프로비저닝 경로 반드시 일치시킬 것
- 대시보드 JSON 파일은 `provisioning/dashboards/` 하위에 위치해야 자동 로드됨
