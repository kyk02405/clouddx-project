# Tutum 시연 영상 설명 정리

## 문서 목적

이 문서는 Tutum 프로젝트 시연 영상을 촬영할 때 바로 활용할 수 있도록 만든 설명 자료입니다.

- 로그인 전 홈페이지에서 무엇을 보여주고 왜 그렇게 기획했는지
- 우리 서비스가 어떤 문제를 해결하는지
- 핵심 기능을 어떤 흐름으로 설명하면 자연스러운지
- `/admin` 모니터링 페이지에서 무엇을 보여주고 왜 중요한지

를 한 번에 정리합니다.

---

## 1. 서비스 한 줄 소개

Tutum은 주식과 코인처럼 여러 곳에 흩어진 자산 정보를 한곳으로 모으고, OCR과 AI를 활용해 입력과 분석의 불편을 줄여주는 AI 자산 관리 서비스입니다.

짧게 말하면:

> "복잡하게 흩어진 자산을 쉽게 등록하고, 실시간 시세와 뉴스, AI 인사이트까지 한 번에 제공하는 개인 자산 관리 플랫폼입니다."

---

## 2. 기획 의도

### 2-1. 사용자가 겪는 문제

기존 자산 관리에서 사용자가 자주 겪는 문제는 크게 네 가지입니다.

1. 국내 주식, 해외 주식, 코인 자산이 각각 다른 앱과 거래소에 흩어져 있습니다.
2. 자산 내역을 직접 정리하려면 종목명, 수량, 평단가를 손으로 입력해야 해서 번거롭습니다.
3. 현재 수익률은 볼 수 있어도, 내 포트폴리오 전체가 어떤 상태인지 직관적으로 이해하기 어렵습니다.
4. 내가 보유한 종목과 직접 연관된 뉴스나 시그널을 한눈에 받기 어렵습니다.

### 2-2. 그래서 Tutum이 해결하려는 것

Tutum은 이 문제를 다음 방식으로 해결하려고 기획했습니다.

- 자산 정보를 한 화면에 통합해 보여준다.
- OCR로 입력 과정을 줄여 사용자의 진입 장벽을 낮춘다.
- AI가 단순 숫자 나열이 아니라 해석까지 도와준다.
- 포트폴리오와 연결된 뉴스, 알림, 분석을 제공한다.
- 운영자 입장에서도 서비스 상태를 빠르게 확인할 수 있게 별도 모니터링 페이지를 제공한다.

즉, Tutum은 단순한 자산 기록 앱이 아니라:

> "입력은 쉽게, 해석은 똑똑하게, 운영은 안정적으로"를 목표로 만든 서비스입니다.

---

## 3. 로그인 전 홈페이지에서 설명할 핵심 메시지

로그인 전 홈페이지는 단순 랜딩 페이지가 아니라, 사용자가 "이 서비스가 어떤 문제를 해결하는지"를 빠르게 이해하게 만드는 역할을 합니다.

설명 포인트는 아래 3개로 잡으면 좋습니다.

1. 흩어진 자산을 한 곳으로 모아준다.
2. OCR과 AI로 입력과 분석을 쉽게 만든다.
3. 실시간 시세, 뉴스, 알림까지 연결해준다.

---

## 4. 로그인 전 홈페이지 설명 흐름

## 4-1. 첫 화면 Hero 섹션

### 화면에서 보여주는 것

- Tutum 브랜드 메시지
- "흩어진 자산을 하나로 모은다"는 핵심 카피
- AI 기반 자산 관리라는 정체성
- OCR, 통합 포트폴리오, 분석 흐름을 암시하는 비주얼

### 설명 포인트

이 구간에서는 서비스의 정체성을 먼저 설명하면 됩니다.

### 멘트 예시

> "Tutum은 주식과 코인처럼 여러 플랫폼에 흩어진 자산을 한 곳에서 관리할 수 있도록 만든 AI 자산 관리 서비스입니다.  
> 사용자는 복잡하게 직접 정리할 필요 없이, 자산을 빠르게 등록하고 현재 상태를 직관적으로 확인할 수 있습니다."

### 강조하면 좋은 점

- 단순 시세 조회 서비스가 아님
- 내 자산을 기준으로 관리와 분석을 같이 제공
- 초보자도 쉽게 쓸 수 있게 설계됨

---

## 4-2. 가치 제안 섹션

이 구간은 "왜 Tutum을 써야 하는가"를 설명하는 파트입니다.

### 핵심 메시지

- 언제 어디서나 실시간 확인 가능
- AI가 데이터를 해석해줌
- 자산 등록이 쉽고 빠름

### 멘트 예시

> "Tutum은 단순히 데이터를 보여주는 것이 아니라, 사용자가 자산을 이해하고 관리하기 쉽게 만드는 데 초점을 맞췄습니다.  
> 실시간으로 자산 상태를 확인할 수 있고, AI가 포트폴리오 흐름을 해석해주며, OCR을 통해 입력 과정도 간소화했습니다."

---

## 4-3. Market Insights / AI 브리핑 영역

### 화면에서 보여주는 것

- 현재 시장 분위기 요약
- 주요 흐름을 짧은 카드 형태로 정리
- 사용자가 복잡한 뉴스 전체를 읽지 않아도 핵심만 파악할 수 있는 구조

### 설명 포인트

여기서는 "정보의 양"이 아니라 "이해 가능한 요약"을 강조하면 좋습니다.

### 멘트 예시

> "시장 뉴스는 많지만, 사용자가 그걸 매번 해석하기는 어렵습니다.  
> 그래서 Tutum은 핵심 시장 흐름을 짧은 인사이트 카드 형태로 정리해서, 사용자가 빠르게 현재 분위기를 파악할 수 있게 구성했습니다."

---

## 4-4. 실시간 상태 바 / 시장 지표 영역

### 화면에서 보여주는 것

- 시세 업데이트 상태
- 뉴스 반영 시점
- AI 스냅샷 반영 시점
- 대표 시장 지표 또는 주요 시장 카드

### 설명 포인트

이 영역은 "서비스가 정적인 정보가 아니라 계속 살아 있는 데이터 흐름 위에서 동작한다"는 걸 보여주는 파트입니다.

### 멘트 예시

> "이 영역은 Tutum이 실시간 데이터 기반으로 동작한다는 점을 보여줍니다.  
> 단순히 과거 데이터를 정리하는 것이 아니라, 현재 시세와 업데이트 흐름을 계속 반영하고 있다는 점이 중요합니다."

---

## 4-5. 뉴스 섹션

### 화면에서 보여주는 것

- 최신 뉴스 목록
- 사용자 관심 자산 또는 포트폴리오 연관 뉴스
- 실시간 수집 기반 뉴스 피드

### 설명 포인트

여기서는 "뉴스를 그냥 많이 보여주는 것"이 아니라 "내 자산과 연결해서 보여준다"는 점을 강조하면 좋습니다.

### 멘트 예시

> "사용자가 모든 뉴스를 일일이 볼 필요는 없습니다.  
> Tutum은 최신 뉴스를 보여주되, 포트폴리오와 연결되는 자산 중심으로 더 의미 있는 정보를 볼 수 있게 설계했습니다."

---

## 4-6. 알림 / Watchlist / 추적 기능

### 화면에서 보여주는 것

- 관심 자산 추적
- 가격 변동 알림
- 중요 이슈를 놓치지 않도록 도와주는 알림 경험

### 설명 포인트

이 부분은 "수동 확인"에서 "능동적인 대응"으로 넘어가는 기능이라고 설명하면 좋습니다.

### 멘트 예시

> "Tutum은 사용자가 매번 직접 확인하는 방식에서 끝나지 않고, 관심 자산이나 중요한 변동을 계속 추적할 수 있게 해줍니다.  
> 즉, 보는 서비스가 아니라 대응할 수 있게 도와주는 서비스라고 설명할 수 있습니다."

## 5. 핵심 기능 설명 정리

영상에서 기능을 소개할 때는 아래 순서가 가장 자연스럽습니다.

## 5-1. 자산 통합 관리

- 국내 주식, 해외 주식, 코인을 한 화면에서 관리
- 자산별 수익률, 평가금액, 비중을 직관적으로 확인
- 포트폴리오 전체 상태를 한 번에 파악 가능

### 멘트 예시

> "Tutum의 핵심은 흩어진 자산을 하나의 포트폴리오 관점에서 통합해 보여준다는 점입니다."

## 5-2. OCR 자동 등록

- 캡처 이미지 업로드로 자산 정보 추출
- 종목명, 수량, 평단가 등 수기 입력 부담 감소
- 사용자 입장에서 가장 진입 장벽이 낮은 기능

### 멘트 예시

> "사용자는 복잡한 자산 정보를 직접 입력하지 않아도 되고, OCR을 통해 빠르게 등록할 수 있습니다."

## 5-3. AI 포트폴리오 인사이트

- 포트폴리오 상태 요약
- 상위 수익 자산, 손실 자산, 현금 비중 등 해석
- 숫자 나열이 아니라 '무슨 의미인지' 설명

### 멘트 예시

> "단순히 수익률을 보여주는 게 아니라, 지금 내 포트폴리오가 어떤 상태인지 AI가 요약해준다는 점이 차별점입니다."

## 5-4. AI 채팅

- 금융 관련 질문 가능
- 포트폴리오, 뉴스, 시장 흐름을 참고한 응답
- 사용자가 자연어로 서비스와 상호작용 가능

### 멘트 예시

> "사용자는 메뉴를 일일이 찾지 않고도, AI에게 질문하듯 필요한 정보를 확인할 수 있습니다."

## 5-5. 개인화 뉴스와 알림

- 포트폴리오 기반 뉴스 추천
- 관심 자산 중심 정보 탐색
- 중요한 변화에 대한 알림 경험

### 멘트 예시

> "내가 실제로 가진 자산과 관련 있는 정보를 중심으로 받아본다는 점에서 개인화된 경험을 제공합니다."

## 5-6. 차트와 시세 확인

- 실시간 가격 흐름
- 자산별 추이 확인
- 포트폴리오 변화 시각화

### 멘트 예시

> "Tutum은 기록만 하는 도구가 아니라, 현재 시장 흐름을 보고 판단할 수 있는 시각화 도구 역할도 합니다."

## 6. `/admin` 모니터링 페이지를 왜 보여줘야 하는가

이 페이지는 일반 사용자용 화면이 아니라 운영자 관점의 관리 콘솔입니다.

시연에서 이 페이지를 보여주는 이유는 두 가지입니다.

1. 우리 서비스가 단순 프론트 결과물만 있는 게 아니라, 실제 운영 가능한 구조를 갖고 있다는 점
2. 데이터 수집, 분석, AI 기능, 백업, 장애 대응까지 고려한 서비스라는 점

즉, 이 페이지는:

> "서비스를 만드는 것"에서 끝나지 않고  
> "서비스를 안정적으로 운영하는 것"까지 생각했다는 근거입니다.

## 7. 모니터링 페이지에서 보여주는 내용

## 7-1. Overview 탭

### 무엇을 보여주는가

- RPS
- P95 응답속도
- Error Rate
- Kafka Lag
- 최근 1시간 API 처리량과 응답속도 추이
- 주요 요약 카드

### 설명 포인트

이 탭은 전체 서비스 상태를 한눈에 보는 요약 대시보드입니다.

### 멘트 예시

> "Overview 탭은 운영자가 가장 먼저 보는 화면입니다.  
> 현재 트래픽이 어느 정도인지, 응답속도가 느려지지는 않았는지, 에러율이 올라가지는 않았는지, 그리고 데이터 파이프라인 지연은 없는지를 한눈에 볼 수 있습니다."

### 왜 중요한가

- 장애 발생 여부를 빠르게 감지할 수 있음
- 사용자 체감 품질과 가장 가까운 지표를 보여줌

---

## 7-2. Infra 탭

### 무엇을 보여주는가

- 노드 상태
- 파드 상태
- CPU / Memory 사용량
- 네임스페이스별 파드 분포
- 인프라 레벨의 이상 여부

### 설명 포인트

이 탭은 서비스가 돌아가는 기반 인프라가 건강한지 보는 곳입니다.

### 멘트 예시

> "Infra 탭에서는 애플리케이션 이전에 클러스터 자체가 안정적인지 확인합니다.  
> 노드 자원이 부족하거나 특정 파드가 재시작을 반복하면, 서비스 문제로 이어질 수 있기 때문에 먼저 이 레벨을 점검합니다."

---

## 7-3. Pipeline 탭

### 무엇을 보여주는가

- 뉴스 파이프라인 상태
- 시세 파이프라인 상태
- 워커별 상태
- 수집, 저장, 인덱싱 흐름
- 각 단계의 문제 요약과 AI 분석

### 설명 포인트

Tutum은 데이터가 계속 들어와야 의미가 있는 서비스이기 때문에, 파이프라인 상태가 매우 중요합니다.

### 멘트 예시

> "Tutum은 뉴스와 시세를 계속 수집하고 저장해야 하는 구조이기 때문에, 파이프라인이 멈추면 서비스 가치가 떨어집니다.  
> 이 탭에서는 producer, consumer, indexing 흐름이 정상적으로 이어지고 있는지를 확인할 수 있습니다."

### 강조 포인트

- 단순한 웹 서비스가 아니라 데이터 파이프라인 기반 서비스
- 운영자 관점에서 가장 실무적인 탭 중 하나

---

## 7-4. Data 탭

### 무엇을 보여주는가

- MongoDB 상태
- Redis 상태
- Kafka 상태
- Elasticsearch 상태
- 디스크 사용량

### 설명 포인트

이 탭은 우리 서비스의 데이터 저장소와 메시지 브로커가 정상적인지 보는 탭입니다.

### 멘트 예시

> "이 탭에서는 서비스의 데이터 레이어를 확인합니다.  
> 단순히 웹 서버가 살아 있는지뿐 아니라, 실제로 데이터가 저장되고 조회되는 기반이 정상인지 확인하는 역할을 합니다."

### 왜 중요한가

- 데이터가 쌓이지 않으면 AI 분석과 뉴스 추천 품질 저하
- 저장소 이상은 서비스 전체 신뢰도에 직접 영향

---

## 7-5. Backup 탭

### 무엇을 보여주는가

- 백업 CronJob 상태
- 최근 실행 여부
- 마지막 성공 여부
- 실패 여부

### 설명 포인트

이 탭은 장애 예방보다도 장애 이후 복구 가능성을 보여주는 탭입니다.

### 멘트 예시

> "운영에서 중요한 건 장애를 막는 것만이 아니라, 문제가 생겼을 때 복구할 수 있는가입니다.  
> Backup 탭은 데이터 보호 관점에서 서비스 운영 성숙도를 보여주는 지점입니다."

---

## 7-6. Logs 탭

### 무엇을 보여주는가

- 최근 로그
- namespace / pod / level 필터
- 에러 요약
- severity 구분
- trace 링크 가능 여부

### 설명 포인트

이 탭은 실제 장애 징후를 가장 가까이서 보는 화면입니다.

### 멘트 예시

> "메트릭이 이상하다는 걸 알았다면, 그다음은 왜 그런지 봐야 합니다.  
> Logs 탭에서는 최근 에러 로그를 직접 확인하고, 어떤 서비스에서 어떤 패턴의 문제가 발생했는지 빠르게 추적할 수 있습니다."

### 강조 포인트

- 단순 로그 출력이 아니라 필터와 중요도 구분이 가능
- 운영 조사 시작점 역할

---

## 7-7. Traces 탭

### 무엇을 보여주는가

- 5xx 서버 에러 트레이스
- 4xx 클라이언트 에러 트레이스
- 느린 요청 트레이스
- 응답 지연 구간 확인

### 설명 포인트

로그가 "무슨 문제가 생겼는지"를 보여준다면, 트레이스는 "어디서 느려졌는지"를 보여줍니다.

### 멘트 예시

> "Traces 탭은 요청이 실제로 어떤 경로를 타고 처리됐는지를 보여줍니다.  
> 즉, 단순히 에러가 났다는 사실을 넘어서, 어느 구간에서 병목이나 지연이 발생했는지 분석할 수 있습니다."

---

## 7-8. AI 분석 기능

### 무엇을 보여주는가

- Overview AI 분석
- Pipeline AI 분석
- Data / Logs / Traces / Backup 진단
- 상태를 요약하고 우선 조치 포인트를 제안

### 설명 포인트

운영자는 수많은 지표를 봐야 하기 때문에, AI 분석은 "대신 판단해준다"기보다 "빠르게 해석을 보조해준다"는 의미로 설명하는 게 좋습니다.

### 멘트 예시

> "운영자는 데이터를 보는 것만으로 끝나지 않고, 무엇부터 조치해야 하는지 판단해야 합니다.  
> Tutum의 Admin AI는 여러 지표를 종합해서 현재 상태를 요약하고, 우선순위가 높은 이슈를 빠르게 파악할 수 있도록 도와줍니다."

## 8. 시연 영상 추천 흐름

아래 순서대로 가면 가장 자연스럽습니다.

1. 로그인 전 홈페이지 진입
2. 서비스 한 줄 소개
3. 기획 의도 설명
4. Hero 섹션과 핵심 가치 설명

7. 마지막에 `/admin` 모니터링 페이지로 이동
8. 운영 관점에서 Overview -> Pipeline -> Logs/Traces 순서로 설명
9. "사용자 경험 + 운영 안정성" 두 축으로 마무리

## 9. 발표용 예시 멘트

## 9-1. 오프닝

> "안녕하세요. 저희 팀이 만든 Tutum은 흩어진 금융 자산을 AI 기반으로 통합 관리할 수 있는 서비스입니다.  
> 사용자는 자산을 쉽게 등록하고, 실시간 시세와 관련 뉴스, 포트폴리오 분석을 한 번에 받을 수 있습니다."

## 9-2. 홈페이지 설명

> "첫 화면에서는 Tutum이 어떤 문제를 해결하는 서비스인지 보여줍니다.  
> 주식과 코인처럼 흩어진 자산을 하나로 모으고, OCR과 AI를 통해 입력과 분석의 불편을 줄이는 것이 핵심입니다."

## 9-3. 기능 설명

> "사용자는 자산을 직접 입력할 수도 있고, 캡처 이미지를 올려 OCR로 빠르게 등록할 수도 있습니다.  
> 이후에는 포트폴리오 현황, AI 인사이트, 관련 뉴스, 차트, 알림까지 연결된 경험을 제공받게 됩니다."

## 9-4. 모니터링 페이지 설명

> "또한 저희는 사용자 기능뿐 아니라 운영 안정성도 중요하게 봤습니다.  
> 이 Admin 페이지에서는 현재 서비스의 응답속도, 에러율, 데이터 파이프라인, 저장소 상태, 로그와 트레이스까지 한 번에 확인할 수 있습니다."

## 9-5. 마무리

> "정리하면 Tutum은 단순히 자산을 기록하는 앱이 아니라,  
> 사용자가 자산을 더 쉽게 이해하고 관리하도록 돕는 AI 자산 관리 플랫폼이며,  
> 동시에 실제 운영 환경까지 고려한 서비스입니다."
