## 2. 상세 변경 사항
- `backend/app/routers/news.py`
  - `html.unescape()`를 적용해 제목, 본문, 소스, 섹션 문자열의 HTML entity를 일반 문자로 복원하도록 수정했다.
  - `_normalize_text()`에 깨진 UTF-8 문자열을 복구하는 방어 로직을 추가했다.
  - `_news_body_from_doc()`도 동일한 정규화 함수를 타도록 바꿔 제목과 본문 처리 기준을 일치시켰다.

## 3. 작업 중 발생 이슈 및 대응
- 이슈: 브라우저 화면에서는 `&quot;`, `&amp;`만 주로 보였지만, 프록시 응답 확인 시 일부 데이터는 한글 자체가 깨진 형태로 섞여 있었다.
- 대응: 단순 HTML entity decode만으로는 부족하다고 판단해, 백엔드 응답 직전 단계에서 mojibake 복구와 entity decode를 함께 수행하도록 보강했다.
