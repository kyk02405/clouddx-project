## 2. 모니터링 대시보드

### 2-1. Overview — 에러 지표

- 에러율을 **% 말고 건수(Count)** 로 표시
- 어떤 에러가 어디서 발생했는지 보여줄 것
- **시계열 그래프** 필요: x축(시간) / y축(에러 건수), 어느 시간에 어떤 에러가 발생했는지

### 2-2. Traces

- 어떤 경로로 trace가 흘렀는지, **어디서 끊겼는지** 보여줘야 함
- 개발자 입장에서 가시성 있게 전달되어야 디버깅이 가능

### 2-3. Kafka Lag

- Kafka lag가 **왜 중요한지 근거**가 있어야 함
- 발표 때 "이게 왜 필요하냐"고 물어보면 답변할 수 있어야 함
- 멘토님 경험: Redis 클러스터 구조에서 클라이언트 버퍼/lag이 높게 치는 이슈 있었음 → 추가 작업 진행한 사례
- **운영에서 왜 lag 모니터링이 중요한지** 한 번 정리해둘 것

### 2-4. CPU / Memory 임계치 & 알림

- 레이턴시 **100ms 이상이면 심각한 수준**
- CPU/Memory 임계치 설정 후 초과 시 **알림 오는 구조** 필요
- 스크래핑 작업 특성상 순간적으로 CPU를 많이 사용하는 케이스가 있음
  → 해당 케이스는 **예외 처리(임계치 제외)** 로 관리하는 방식도 있음

## 6. 보안

### OS 수준 방화벽

- Ubuntu 22 기준 `/etc/hosts.allow` / `/etc/hosts.deny` 로 SSH 접근 IP 제한
  - `hosts.deny`에 `ALL:ALL` 전체 차단 후 팀원 IP만 `hosts.allow`에 추가
- `fail2ban` 라이브러리로 해외 IP / 무차별 접근(brute force) 차단
- `/var/log/auth.log` 확인 시 **중국 IP 접근 시도 이력** 있을 수 있음 → 확인 및 어필 포인트
- SSH `PermitRootLogin no` 설정 확인 (`/etc/ssh/sshd_config`)
- **비밀번호 정책** 설정 (특수문자, 최소 길이 등)

### Ubuntu EOL 대비

- 현재 Ubuntu 22.04 사용 중 → EOL 2027년 4월 예정
- EOL 이후 보안 업데이트 중단 → 업그레이드 계획 수립 필요

### PPT 어필 포인트

> "내부 방화벽은 Ubuntu `hosts.allow/deny`로 설정,
> 외부 방화벽은 AWS WAF로 설정했다" 처럼
> **내부/외부 보안을 나눠서 설명**하면 신경 쓴 티가 남

- KISA 보안 점검 항목 참고해서 체크리스트 작성해두면 좋음
