# 2026-02-13 개발 로그: 뉴스 로딩 및 소셜 계정 탈퇴 이슈 해결

## 1. 이슈 요약

- **뉴스 로딩 이슈**: 메인 페이지 뉴스 미출력 및 대시보드 뉴스 섹션 깜빡임(불안정) 현상 발생.
- **계정 탈퇴 이슈**: 구글 소셜 계정 탈퇴 후 재가입 시 "이미 가입된 계정" 오류 발생 (데이터 잔재).

## 2. 작업 내용

### 뉴스 서비스 안정화

- **백엔드**: MongoDB `clouddx.news` 컬렉션 데이터 확인 (2,200+ 건 정상 적재 상태).
- **프론트엔드 프록시 보강**:
  - `frontend/app/api/public/news/route.ts`가 `query` 파라미터를 지원하도록 수정.
  - 백엔드 URL 하드코딩 제거 및 `NEXT_PUBLIC_API_URL` 환경 변수 활용 구조 강화.
- **컴포넌트 수정**:
  - `PersonalizedNewsCarousel.tsx`: 브라우저 직접 fetch(`localhost:8000`)를 제거하고 내부 프록시를 통하도록 수정하여 CORS 및 연결 신뢰성 확보.
  - `NewsSection.tsx`: 백엔드에서 내려주는 실제 페이지네이션 정보를 사용하도록 동적 처리.

### 소셜 계정 탈퇴 로직 보강

- **멀티 DB 정리**: `backend/app/routers/auth.py`의 `withdraw_account` 엔드포인트에서 MariaDB 삭제 후 MongoDB 레거시 `users` 컬렉션의 정보도 함께 삭제하도록 보강.
- **수동 정리**: 재가입이 차단되었던 사용(`rubyjeenkim@gmail.com`)의 MariaDB 및 MongoDB 흔적을 수동 스크립트로 완전 삭제 처리.

## 3. 결과 확인

- **뉴스**: 메인 페이지와 대시보드 모두에서 실시간 뉴스가 끊김 없이 노출됨. 페이지네이션 정상 작동.
- **탈퇴**: 탈퇴 후 모든 DB에서 데이터가 정리되어 동일 이메일로 즉시 재가입 가능함을 검증.

## 4. 향후 계획

- 현재 뉴스는 MongoDB를 소스로 사용하고 있으나, 운영 환경 요구사항에 따라 MariaDB로의 데이터 이관 고려 가능.
