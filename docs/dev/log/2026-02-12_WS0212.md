# 2026-02-12 작업 로그 (WS0212)

## 작업 목적
- Kafka 설정에서 ZooKeeper 의존 제거 및 KRaft 모드 통일
- 시세 조회를 REST 중심에서 WebSocket 실시간 스트림 + REST 폴백 구조로 확장
- 프론트 주요 화면에 스트림 연결 상태 배지 노출

## 변경 요약

### 1) Kafka/KRaft 전환
- `frontend/docker-compose.yml`
  - `zookeeper` 서비스 제거
  - Kafka를 단일 노드 KRaft 설정으로 전환
  - `KAFKA_ZOOKEEPER_CONNECT` 제거
  - `KAFKA_PROCESS_ROLES`, `KAFKA_CONTROLLER_QUORUM_VOTERS`, `CLUSTER_ID` 등 KRaft 필수 설정 추가

### 2) 문서 정리 (ZooKeeper -> KRaft)
- `docs/K8S_MIGRATION_PLAN.md`
  - 아키텍처/리소스/배포 단계 내 ZooKeeper 용어를 KRaft 기준으로 정정
- `docs/work-plans/2026-02-11 Elastic-search-pipeline.md`
  - Kafka 구성 설명을 KRaft 기준으로 정정
- `docs/KAFKA_ADOPTION_PLAN.md` (신규)
  - Kafka 적용 로드맵(토픽/그룹/단계별 계획/운영 체크리스트) 작성

### 3) 백엔드 WebSocket 시세 스트림 추가
- `backend/app/routers/market.py`
  - 신규 엔드포인트: `/api/v1/market/ws`
  - 기능:
    - `symbols`, `interval_ms` 기반 구독
    - `subscribe/unsubscribe/set/ping` 제어 메시지 처리
    - Redis 캐시 우선 조회 + KIS/Upbit API fallback
    - `prices` 이벤트 주기 전송

### 4) 프론트 WebSocket 연동
- `frontend/lib/hooks/useMarketPrices.ts`
  - WebSocket 우선 수신, 실패 시 REST 폴백
  - 자동 재연결 로직 추가
  - 스트림 상태(`connecting`, `connected`, `reconnecting`, `fallback`) 추가

- `frontend/components/WatchlistSidebar.tsx`
  - WS 실시간 수신 연동
  - 폴백 유지(REST)
  - 상태 배지 표시 추가

- `frontend/components/ChartSidebar.tsx`
  - `useMarketPrices`의 스트림 상태를 받아 상태 배지 표시

- `frontend/context/AssetContext.tsx`
  - 보유 자산 심볼 기반 WS 구독
  - 수신 가격으로 holdings 실시간 갱신
  - WS 실패 시 REST 폴백 유지
  - `priceStreamStatus` 컨텍스트 값 추가

- `frontend/app/portfolio/asset/page.tsx`
  - `priceStreamStatus` 기반 상태 배지 표시
